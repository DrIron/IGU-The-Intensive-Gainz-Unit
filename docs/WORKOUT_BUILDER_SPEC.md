# IGU Workout Builder System Specification

## Overview

The IGU Workout Builder is a comprehensive system for coaches to create, manage, and assign workout programs to clients. It features a flexible column-based exercise prescription system, multi-session days, and a calendar-based program structure.

---

## Design Principles

1. **Mobile-First for Clients** - Client workout logging must be thumb-friendly and optimized for gym use on phones
2. **Desktop-Optimized for Coaches** - Program builder can leverage larger screens but must remain functional on tablets
3. **Flexible Prescriptions** - Coaches choose exactly which fields to show per exercise via the column system
4. **Shared Exercise Library** - Site-wide exercise database used by all coaches and clients

### 1.5 Exercise Library (Site-Wide)

A shared, site-wide exercise database used by all coaches and clients.

**Features:**
- Search by name, muscle group, equipment
- Filter by: Primary Muscle, Equipment, Type (Compound/Isolation)
- Admin-managed (coaches can suggest additions)

**Each exercise contains:**
- Name
- Video demo (YouTube link or upload)
- Primary/Secondary muscles
- Equipment required
- Movement pattern
- Default instructions/cues

### Key Rules

1. **Synchronized Position** - All team members are on the same week/day
2. **New Members Start Current** - No catching up; join where the team is now
3. **Private Logs** - Members cannot see each other's logged weights/reps
4. **Shared Program** - Any program changes affect all team members
5. **Individual Swaps** - Exercise swaps are per-member, not team-wide

### Team vs Individual Assignment

| Aspect | Individual Assignment | Team Assignment |
|--------|----------------------|-----------------|
| Start position | Week 1, Day 1 | Team's current position |
| Program pace | Client's own pace | Team's synchronized pace |
| Rest days | Flexible | Fixed team schedule |
| Logs visible to | Coach only | Coach only (private per member) |
| Exercise swaps | Per client | Per client (not team-wide) |

### Key Database Tables

```sql
-- Teams
teams (
  id UUID PRIMARY KEY,
  name VARCHAR(100),
  program_id UUID REFERENCES programs(id),
  current_week INTEGER DEFAULT 1,
  current_day INTEGER DEFAULT 1,
  start_date DATE,
  coach_id UUID REFERENCES coaches(id),
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Team Members
team_members (
  id UUID PRIMARY KEY,
  team_id UUID REFERENCES teams(id),
  client_id UUID REFERENCES clients(id),
  joined_at TIMESTAMPTZ DEFAULT now(),
  status VARCHAR(20) DEFAULT 'active'
);

-- Exercise Swaps
exercise_swaps (
  id UUID PRIMARY KEY,
  client_id UUID REFERENCES clients(id),
  program_assignment_id UUID, -- individual or team assignment
  original_exercise_id UUID REFERENCES exercises(id),
  replacement_exercise_id UUID REFERENCES exercises(id),
  scope VARCHAR(20), -- 'session' or 'plan_wide'
  reason TEXT,
  coach_approved BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Exercise Muscle Contributions (for volume algorithm)
exercise_muscle_contributions (
  id UUID PRIMARY KEY,
  exercise_id UUID REFERENCES exercises(id),
  muscle_group VARCHAR(50),
  contribution_type VARCHAR(20), -- 'primary', 'secondary', 'synergist', 'stabilizer'
  contribution_pct DECIMAL(3,2), -- 0.60 = 60%
  UNIQUE(exercise_id, muscle_group)
);

-- Workout Session Logs (with volume)
workout_logs (
  id UUID PRIMARY KEY,
  client_id UUID REFERENCES clients(id),
  session_id UUID REFERENCES sessions(id),
  scheduled_date DATE,
  completed_at TIMESTAMPTZ,
  total_volume DECIMAL(12,2), -- calculated: sum of (weight Ã— reps)
  total_sets INTEGER,
  total_reps INTEGER,
  duration_minutes INTEGER,
  avg_rpe DECIMAL(3,1),
  session_rating VARCHAR(20), -- 'hard', 'normal', 'easy', 'great'
  notes TEXT
);

-- Exercise Logs (with volume)
exercise_logs (
  id UUID PRIMARY KEY,
  workout_log_id UUID REFERENCES workout_logs(id),
  exercise_id UUID REFERENCES exercises(id),
  exercise_order INTEGER,
  volume DECIMAL(10,2), -- sum of (weight Ã— reps) for this exercise
  created_at TIMESTAMPTZ DEFAULT now()
);
```

## Exercise Editor V2 (Implemented - Phase 18)

### Overview

Replaced the shared-prescription exercise editor (`DynamicExerciseRow`) with a per-set row-based layout (`ExerciseCardV2`). Each set now has its own editable row with independent values. Two visually separated column categories: "Exercise Instructions" (coach prescriptions) and "Client Inputs" (empty fields for client).

### Database Changes

- `sets_json JSONB DEFAULT NULL` added to `exercise_prescriptions` table
- When NULL, legacy scalar fields (`set_count`, `rep_range_min`, etc.) are used
- Migration: `supabase/migrations/20260206_exercise_editor_v2.sql`

### New Data Model

Each exercise prescription stores per-set data as:
```json
{
  "sets_json": [
    { "set_number": 1, "rep_range_min": 8, "rep_range_max": 12, "weight": 50, "rir": 2, "rest_seconds": 90, "tempo": "3010" },
    { "set_number": 2, "rep_range_min": 8, "rep_range_max": 12, "weight": 50, "rir": 2, "rest_seconds": 90, "tempo": "3010" },
    { "set_number": 3, "rep_range_min": 8, "rep_range_max": 12, "weight": 50, "rir": 1, "rest_seconds": 90, "tempo": "3010" }
  ]
}
```

### New Types

- `SetPrescription` â€” per-set values (set_number, reps, rep_range_min/max, weight, tempo, rir, rpe, percent_1rm, rest_seconds, time_seconds, distance_meters, band_resistance, notes, custom_fields)
- `EnhancedExerciseDisplayV2` â€” extends `EnhancedExerciseDisplay` with `sets[]`, `prescription_columns[]`, `input_columns[]`
- `DEFAULT_INPUT_COLUMNS` â€” default client input columns (Weight, Reps, RPE)
- New column types: `band_resistance` (prescription), `performed_hr`/`performed_calories` (client input)

### Helper Functions

- `splitColumnsByCategory(columns)` â€” splits ColumnConfig[] into prescription vs input categories
- `legacyPrescriptionToSets(prescription)` â€” expands shared values into N identical SetPrescription rows
- `getSetColumnValue(set, columnType)` / `setSetColumnValue(set, columnType, value)` â€” per-set getters/setters
- `getYouTubeThumbnailUrl(videoUrl)` â€” extracts YouTube video ID â†’ thumbnail URL
- `reorderColumns(columns, fromIndex, toIndex)` â€” reorder columns within a category

## Column Header Drag-to-Reorder (Implemented - Phase 19)

### Requirement

Column headers are directly draggable in the exercise table for intuitive reordering.

### Behavior

- Drag handle icon (GripVertical) appears on hover over column header
- Dragged column shows as semi-transparent (opacity-40)
- Drop zone highlights with a primary-colored ring
- Two separate reorder zones:
  1. **Exercise Instructions** â€” columns reorder within this group only
  2. **Client Inputs** â€” columns reorder within this group only
- Cannot drag a column from one category to another (enforced by category check)
- New order persists to `column_config` JSONB via the existing save mechanism

### Implementation

Uses native HTML5 drag events on `<th>` elements (matches existing `ColumnConfigDropdown` drag pattern). Each column header has `draggable` attribute and handlers for `onDragStart`, `onDragOver`, `onDrop`, `onDragEnd`.

The `reorderColumns(columns, fromIndex, toIndex)` helper splices the moved column and reassigns `.order` values.

### Files

| File | Change |
|------|--------|
| `ColumnCategoryHeader.tsx` | Added drag state, native drag handlers per category, visual feedback classes |
| `ExerciseCardV2.tsx` | Added `handleReorderPrescriptionColumns` / `handleReorderInputColumns` callbacks |
| `src/types/workout-builder.ts` | Added `reorderColumns()` helper |
| `SetRowEditor.tsx` | No changes needed â€” already sorts by `.order` |
